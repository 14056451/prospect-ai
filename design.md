# ProspectAI - 모바일 앱 인터페이스 설계

## 앱 개요
AI를 활용한 가망 고객 관리 및 상담 요약 기능을 제공하는 모바일 앱. 1인 영업/네트워크마케팅 사업자를 타겟으로 상담 메모 정리와 고객 추적을 간편하게 하는 도구.

---

## 스크린 리스트 (MVP 기준 5개 화면)

| 화면명 | 목적 | 주요 콘텐츠 |
|--------|------|-----------|
| **메인 리스트** | 가망 고객 전체 조회 | 고객 리스트, 다음 연락일, 상태 필터 |
| **고객 상세** | 개별 고객 정보 관리 | 이름, 연락처, 메모, 상담 일시, 상태, 다음 연락일 |
| **상담 메모 입력** | 상담 후 메모 기록 | 메모 텍스트 입력, AI 요약 생성 버튼 |
| **AI 요약 결과** | AI 생성 요약 및 액션 | 요점 3줄, 다음 액션 3개 (프리미엄) |
| **통계 대시보드** | 월별 성과 조회 | 상담 건수, 전환율, 메모 키워드 (프리미엄) |

---

## 주요 사용자 흐름

### 1. 가망 고객 추가 흐름
```
메인 리스트 → [+ 추가] 버튼 → 고객 정보 입력 → 저장 → 메인 리스트 갱신
```

### 2. 상담 기록 흐름
```
메인 리스트 → 고객 선택 → 고객 상세 화면 → [메모 추가] → 메모 입력 → 저장 → AI 요약 생성 (프리미엄)
```

### 3. 다음 연락일 알림 흐름
```
다음 연락일 1일 전 또는 당일 → 앱 내 알림 표시 → 사용자 탭 → 고객 상세로 이동
```

---

## 화면별 상세 설계

### 1. 메인 리스트 화면 (Home)
**목적**: 모든 가망 고객을 한눈에 조회하고 빠르게 액션 취하기

**레이아웃**:
- 상단: 앱 제목 + 알림 아이콘 (다음 연락일 고객 수)
- 중단: 상태 필터 탭 (전체, 1차 상담, 2차, 소비자, 사업자)
- 하단: 고객 리스트 (FlatList)

**각 리스트 아이템**:
```
┌─────────────────────────────┐
│ 이름 (굵음)      다음 연락일  │
│ 전화 | 직업      상태 배지   │
│ 구매: 상품명 × 수량 | 가격   │
└─────────────────────────────┘
```

**상호작용**:
- 아이템 탭 → 고객 상세 화면
- [+ 추가] 버튼 → 고객 추가 모달
- 상태 필터 탭 → 리스트 필터링
- 알림 아이콘 → 다음 연락일 고객 목록

**색상**:
- 상태 배지: 1차(파랑), 2차(주황), 소비자(초록), 사업자(보라)

---

### 2. 고객 상세 화면 (Customer Detail)
**목적**: 개별 고객의 모든 정보 관리 및 상담 기록

**레이아웃**:
- 상단: 고객 이름 + 수정/삭제 버튼
- 중단: 고객 정보 카드 (스크롤 가능)
  - 연락처 (전화번호 클릭 시 전화 앱 연동)
  - 직업/회사명
  - 상태 (드롭다운)
  - 다음 연락일 (자동 계산 또는 수동 입력)
  - 상담 일시
- 하단: 메모 섹션
  - 기존 메모 리스트 (최근순)
  - [+ 메모 추가] 버튼

**상호작용**:
- 상태 드롭다운 변경 → 자동 저장
- 다음 연락일 선택 → 캘린더 피커 → 알림 설정
- [+ 메모 추가] → 메모 입력 화면으로 이동
- 메모 아이템 탭 → 메모 상세/수정 화면

---

### 3. 상담 메모 입력 화면 (Memo Input)
**목적**: 상담 후 메모를 빠르게 기록하고 AI 요약 생성

**레이아웃**:
- 상단: "상담 메모 입력" 제목
- 중단: 텍스트 입력 필드 (다중행)
- 하단: 
  - [저장] 버튼
  - [AI 요약 생성] 버튼 (프리미엄 사용자만 활성)

**상호작용**:
- 메모 입력 → 저장 → 고객 상세 화면으로 돌아가기
- [AI 요약 생성] 클릭 → 로딩 표시 → AI 요약 결과 화면으로 이동

**색상**:
- 저장 버튼: 기본 색상 (파랑)
- AI 요약 버튼: 프리미엄 표시 (금색 또는 강조 색상)

---

### 4. AI 요약 결과 화면 (AI Summary) - 프리미엄
**목적**: AI가 생성한 요약과 다음 액션 표시

**레이아웃**:
- 상단: "AI 요약 결과" 제목
- 중단:
  - 요점 섹션 (3줄)
  - 다음 액션 섹션 (3개 항목)
- 하단:
  - [저장] 버튼
  - [다시 생성] 버튼

**콘텐츠 예시**:
```
요점:
1. 고객이 제품에 관심 있음, 가격 문의 중
2. 다음 주 회의 예정, 최종 결정 예상
3. 추가 자료 필요, 카탈로그 발송 요청

다음 액션:
1. 카탈로그 이메일 발송 (내일)
2. 다음 주 월요일 전화 (최종 확인)
3. 계약서 준비 (승인 대기)
```

**상호작용**:
- [저장] → 메모에 AI 요약 추가 → 고객 상세로 돌아가기
- [다시 생성] → 다시 AI 호출 → 새 요약 표시

---

### 5. 통계 대시보드 (Dashboard) - 프리미엄
**목적**: 월별 영업 성과 시각화

**레이아웃**:
- 상단: 월 선택 (좌우 화살표로 이전/다음 월)
- 중단: 3개 카드
  - 상담 건수 (숫자 + 그래프)
  - 전환율 (백분율 + 진행 바)
  - 많이 사용하는 메모 키워드 (워드 클라우드 또는 리스트)
- 하단: 상담 상태별 분포 (파이 차트)

**색상**:
- 카드 배경: 밝은 회색 (surface)
- 그래프: 브랜드 주요 색상 사용

**상호작용**:
- 월 선택 → 데이터 갱신
- 각 카드 탭 → 상세 정보 (옵션)

## 반응형 레이아웃
- **기본**: 모바일 세로 방향 (9:16)
- **한 손 조작**: 모든 버튼은 화면 하단 50% 이내에 배치
- **안전 영역**: SafeArea 적용 (노치, 홈 인디케이터 고려)

---

## 접근성
- 모든 텍스트는 최소 14px 이상
- 터치 타겟 최소 44x44pt
- 색상만으로 정보 전달 금지 (아이콘/텍스트 함께 사용)
- 다크 모드 자동 지원

---

## 마이크로 인터랙션
- 버튼 탭: 0.97 스케일 + 가벼운 햅틱
- 리스트 스크롤: 부드러운 스크롤
- 로딩: 스피너 표시
- 성공/실패: 토스트 메시지 + 햅틱 피드백
